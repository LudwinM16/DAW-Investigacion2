indexedDB

Es una API de almacenamiento (“Base de datos indexada” en español), brindando la capacidad de guardar información como datos estructurados, similar a una base de datos local en el navegador.

Es importante conocer que es preferible utilizar indexedDB antes que localStorage, pues la primera ofrece opciones de búsqueda y filtrado (como una base de datos) siendo útiles si se almacena un gran cúmulo de información y mejoran grandemente el rendimiento de una aplicación; en cambio, la segunda, almacena la información en formato “clave-valor”, en dado caso que se quisiera almacenar un listado, habría que realizarlo al interior del navegador y en cada consulta obtener dichos datos en la memoria y hacer las operaciones pertinentes; mientras que con indexedDB, todas estas operaciones serían manejadas por el navegador y solo se obtendría el resultado de las consultas.

Métodos/Procesos de indexedDB

Métodos



open()

Brinda acceso de manera asincrónica a una base de datos, se le debe ingresar un nombre o identificador y opcionalmente, una versión.

IDBFactory

Da acceso a la base de datos, siendo la interfaz global de indexedDB, además de ser su punto de acceso.

IDBCursor

Realiza iteraciones o navega a través de los objetos de almacenamiento o índices.

IDBCursorWithValue

Al igual que IDBCursor(), navega a través de los objetos almacenados pero además devuelve el valor actual del objeto en que se ha ubicado el “cursor”.

IDBDatabase

Es una representación de la conexión con la base de datos y siendo la única manera de hacer transacciones con ella.

IDBEnvironment

Permite acceder a la base de datos para el front-end o lado del cliente, se usa como parte del objeto Window.

IDBIndex

Accede a los metadatos de un índice.

IDBKeyRange

Establece un rango de claves o propiedades almacenadas.

IDBObjectStore

Representa un objeto para almacenar datos o información.

IDBOpenDBRequest

Indica condiciones para ingresar a una base de datos.

IDBRequest

Obtiene los resultados de una condición a la base de datos y a sus objetos.

IDBTransaction

Indica una transacción o consulta a la base de datos, se debe indicar a qué objeto se va a acceder y el tipo de acceso deseado (de lectura o escritura).

IDBVersionChangeEvent

Permite saber sí la versión de la base de datos actual ha cambiado.



Ventajas Y Desventajas

Ventaja
Permite trabajar con una gran cantidad de datos de manera fácil y eficiente.

Ventaja
Se puede trabajar de forma organizada haciendo uso de Promises.

Desventaja
Su nivel de complejidad es mayor respecto a utilizar y configurarlo en comparación a localStorage y sessionStorage.

Desventaja
Una petición fallida automáticamente aborta la transacción, cancelando todos sus cambios.
