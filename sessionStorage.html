sessionStorage

Es una propiedad del objeto Window que nos permite acceder al objeto Storage (“Almacenamiento de sesión” sería el nombre en español de esta propiedad), siendo similar a un Objeto, el cual guarda información a través del formato "clave-valor", esta información tiene vigencia en una página mientras esté abierta en una ventana del navegador, perdurando aún si se recarga y/o se vuelve abrir dicha página en la misma ventana.

Sin embargo, al abrir la página en una nueva ventana del navegador, se iniciará una nueva sesión sin la información almacenada. Cabe mencionar que sessionStorage también se puede denominar como un objeto de almacenamiento web dado a que acceden a un objeto Storage.

La capacidad de almacenamiento de este objeto oscila entre los 5 y 10 MB, según el navegador.

Si se desea acceder a los datos almacenados, se debe hacer mediante scripts del mismo origen.

Cabe mencionar que mayoritariamente la información almacenada, se guarda como tipo texto y su uso es limitado dentro de la misma página.

- Objeto Storage: Es un objeto que permite almacenar información para un documento web específico, almacenando dicha información en una sesión (sessionStorage) o de manera local (localStorage), permitiendo añadir, modificar o eliminar los datos almacenados.

- Formato “clave-valor”: siendo sessionStorage similar a un objeto, contiene pares de clave-valor (propiedades). Es decir, que un objeto puede contener propiedades y cada una de estas tiene un nombre único también llamado clave, que tiene un valor respectivo.

Métodos y propiedades de sessionStorage

Teniendo en cuenta que sessionStorage accede a un objeto Storage, dicho objeto hereda las siguientes propiedad y métodos:

Propiedades



length

-> Determina la cantidad de elementos almacenados en el objeto, devolviendo dicha cantidad como un número entero.

Métodos



key()

-> Este método requiere de un número entero “n” para acceder al enésimo elemento almacenado

getItem()

-> Se debe ingresar el nombre de una clave (elemento o propiedad) para que este método devuelva el valor de dicho elemento almacenado en el Objeto

setItem()

-> Este método necesita un nombre y un valor para añadir en el Objeto una propiedad con ese nombre como su nombre o clave y cuyo valor será el ingresado, si ya existía dicha propiedad sólo actualiza su valor. Para ingresar los campos necesarios, se deben separar por comas ( , ) sessionStorage(clave, valor);

removeItem()

-> Requiere que se ingrese el nombre de una propiedad para que este método la elimine del objeto

clear()

-> Al llamar este método, se vacían el objeto Storage, eliminando todas las propiedades almacenadas y su valor o información contenida.

Ventajas y desventajas

Estas son algunas ventajas y desventajas que sessionStorage tiene:

Ventaja

Accede a la información de manera rápida con el formato "clave-valor"

Ventaja

Cuenta con capacidad de almacenar desde 5 hasta 10MB

Desventaja

Al cerrar la ventana en el navegador, la información se pierde.

Desventaja

Los datos obtenidos no se puede compartir entre mas de una ventana.
